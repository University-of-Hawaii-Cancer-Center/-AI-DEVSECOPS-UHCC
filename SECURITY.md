# Security Policy

## Supported Versions

Only the current major version is supported with security updates. Please ensure you are running the latest version of the CI/CD pipeline before submitting a report.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a top priority for the University of Hawaii Cancer Center (UHCC) and the Research Corporation of the University of Hawaii (RCUH). If you discover a vulnerability or security flaw in this repository—especially one that could expose Protected Health Information (PHI) or bypass NIST 800-53 controls—please report it immediately.

**Do NOT open a public GitHub issue.**

Please email your findings directly to the UHCC Information System Security Manager (ISSM) and the security team at:
**uhcc-security@hawaii.edu**

### What to include
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact (e.g., Remote Code Execution, Privilege Escalation, Secret Leakage).

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. If you do not receive a reply, please reach out to the RCUH compliance hotline.
